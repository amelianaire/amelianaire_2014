Design Language & Pattern Library

I proposed and led the creation of a design language. Collaborating with my teammates on the updated look and feel, we created our company’s first pattern library.

Goals

  • Upgrade the site's visual styling to effectively communicate the brand
  • Create a color palette meeting AA standards for accessibility.
  • Collaborate with the front-end developers to build a living pattern library with sustainable coding practices.

Process

Step 1

Compiled a UI inventory, sorting elements into 3 piles: keep, discard, or merge. Collaborated with front-end developers for naming conventions of patterns we kept.

Screenshots from the UI inventory exercise

Step 2

Created a list of brand characteristics to drive visual concepts. Each designer then created a Style Tile to present to the Marketing SVP. View my full Style Tile.

My Style Tile concept

Step 3

Converged styles and refined color palette based on SVP feedback. Split up UI Pattern List among three designers, and iterated on our respective sections based on our approved visual direction, and got developer buy-off on patterns as we went.

New color palette with 8 core values, down from 43 previously. All colors take into account WCAG AA standards for contrast.

Step 4

Tested new patterns in new mobile site design, correcting as needed. I created an Axure widget library for new styles for quick implementation in high-fidelity comps. We've recently started introducing select styles to desktop as time allows. (Very slow process!)

Step 5

After doing a couple card sorts to determine organization, we are introducing completed patterns into our official pattern library, hosted on Frontify. We'll be introducing it into the development process in Spring 2018.
